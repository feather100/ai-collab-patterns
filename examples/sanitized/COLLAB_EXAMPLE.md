# 【项目名】协作规范（示例）

> 本文档展示协作规范的格式，所有内容均为占位演示。
>
> 最后巡检日期: YYYY-MM-DD

---

## 1. 为什么需要这份规范

**多 AI 协同最大的风险是不一致与不可追溯。**

## 2. 唯一事实源（SoT）与一致性铁律

| 层 | 文件 | 谁说了算 |
|---|---|---|
| **标准** | `【路径】` | **以此为准** |
| **决策记录** | `【路径】` | 每条决策为什么这么定 |

**一致性铁律：** 改 SoT ⟺ 更新决策记录 ⟺ 下游自检通过。

## 3. 入会检查

1. 读 AI_ENTRY.md + 本文件
2. 读决策记录
3. 先改 SoT，再改下游

## 4. 变更流程

```
读 SoT → 确定层级 → 写决策记录 → 改 SoT → 改下游 → 验收 → 留痕 → 提交
```

## 5. 验收标准

- [ ] 核心自检通过
- [ ] SoT 与决策记录已同步
- [ ] 改动已在 docs/ 留痕
- [ ] 未引入未经许可的外部 API 调用
