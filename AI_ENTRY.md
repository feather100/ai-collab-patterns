## 硬性约束（违反即不予验收）

这些约束**不可绕过**，任何 AI 都不得以"没看到"为由违反：

1. **📝 文档留痕** — 任何设计决策都要在决策记录中写明理由。改了标准必须先追加决策记录、再改 SoT。
2. **💰 成本确认** — 任何**产生费用 / 调外部 API / 拉大体量内容**的动作，执行前先停下说明开销，等人类许可，**绝不默认执行**。真花钱的代码路径必须带硬性成本闸。
3. **🎯 SoT 一致** — 改 SoT 必同步决策记录和下游实现，三者不一致即视为未完成。
4. **🔒 默认零成本路径** — 验证走离线（mock）路径，真调外部服务永远是显式二次确认的选择。
5. **📅 文档巡检** — 所有核心文档必须定期或在重大架构变更后进行巡检。巡检后需在文档头部标注 `> 最后巡检日期: YYYY-MM-DD`。
